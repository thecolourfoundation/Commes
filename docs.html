Commes Documentation

A quantum-resistant, AI-powered security layer for Node.js applications. One middleware call. Six layers of protection.

v1.0.0 — Now on npm
Commes is production-ready. Install with npm install commes.

Commes combines platform-level threat detection with post-quantum cryptographic primitives — all surfaced through an AI agent that explains every attack in plain English while your application runs.

It detects SQL injection, NoSQL injection, command injection, XSS, path traversal, prototype pollution, CSRF, and secret leakage — before any of it reaches your handlers.

Installation

Install Commes from npm. Node.js 18 or later is required.

bash 
npm install commes

Commes ships its own WASM crypto primitives with zero peer dependencies. It works in Node.js 18+ and Bun.

Quick start

Drop shield() into your middleware stack as early as possible. Parse request bodies before Commes so it can inspect them.

javascript 
import express from 'express';
import { shield } from 'commes';

const app = express();

// Parse bodies before Commes so it can inspect them
app.use(express.json({ limit: '1mb' }));
app.use(express.urlencoded({ extended: true }));

// Drop this in as early as possible
app.use(shield({
  rateLimit: {
    windowMs: 60_000,
    maxRequests: 100,
    blockDurationMs: 300_000,
  },
  whitelist: ['/healthz'],
  dryRun: false,
}));

app.get('/', (req, res) => {
  res.json({ status: 'ok' });
});

app.listen(3000);

Requirements

  • Node.js 18 or later
  • Bun (supported)
  • TypeScript 5+ (optional — full types included)

Shield middleware

The shield() function returns an Express-compatible middleware that inspects every incoming request for threats before it reaches your route handlers.

What it detects:

  • SQL injection, NoSQL injection, command injection
  • Cross-site scripting (XSS)
  • Path traversal attacks
  • Prototype pollution
  • CSRF attempts
  • Secret leakage in request bodies
  • Behavioural anomalies vs. your traffic baseline

Every response also gets the full OWASP-recommended security header suite: strict CSP, HSTS, X-Frame-Options, Permissions-Policy, and Cross-Origin policies.

Threat events

Commes emits structured events you can forward to any logging pipeline, Slack, or PagerDuty.

javascript 
import { shieldEmitter } from 'commes';

// Every detected threat, blocked or flagged
shieldEmitter.on('threat', (event) => {
  console.log(`[${event.severity}] ${event.category} from ${event.source.ip}`);
});

// Only requests that were actually blocked
shieldEmitter.on('blocked', (event) => {
  myLogger.warn('Request blocked', { event });
});

AI agent

The interactive CLI agent watches your event stream, prints structured threat reports, correlates events across IPs, detects multi-vector attack patterns, and answers your questions about what's happening.

javascript 
import { Agent } from 'commes';

const agent = new Agent({
  verbosity: 'verbose',   // 'concise' for one-liners
  streamEvents: true,
  color: true,
});

agent.start(true); // true = open interactive REPL

REPL commands

CommandDescription
statusOverall threat statistics and uptime
threatsThe 10 most recent threat events
trafficRequest rate by endpoint

clearClear the terminal
exitShut down the agent

Examples: explain xss · explain sql-injection · explain prototype-pollution

Rate limiting
