How it works

Three lines.
Total protection.

server.js
// 1 — install
import commes from 'commes';

// 2 — configure
const guard = commes({
  crypto: 'ml-kem-768',
  ai: true,
  explain: 'en',
  webhook: process.env.SLACK_URL
});

// 3 — protect
app.use(guard);
01

Install via npm

One package, zero peer dependencies. Works in Node 18+ and Bun. Commes ships its own WASM crypto primitives.

02

Configure in seconds

Pick your key algorithm, enable the AI classifier, and point alerts at Slack, PagerDuty, or any webhook URL.

03

Every request is protected

Commes inspects, classifies, and cryptographically signs every response before it leaves your server — in under a millisecond.

04

Read the explanation

When something's blocked, the AI agent writes a one-paragraph brief: what the attacker attempted, why it failed, what to watch for next.

Cryptography

NIST-approved.
Quantum-proof.

Commes implements all three NIST Post-Quantum Cryptography standards — the only suite that's secure against both classical and quantum adversaries.

Key Encapsulation

ML-KEM

Module-Lattice-based Key Encapsulation — formerly known as CRYSTALS-Kyber. Replaces RSA and ECDH for key exchange.

StandardFIPS 203
Security levelCategory 3 (AES-192)
Public key size1,184 bytes
Ciphertext size1,088 bytes

Digital Signatures

ML-DSA

Module-Lattice-based Digital Signature Algorithm — formerly CRYSTALS-Dilithium. Replaces ECDSA for signing.

StandardFIPS 204
Security levelCategory 3 (AES-192)
Signature size3,293 bytes
Public key size1,952 bytes
